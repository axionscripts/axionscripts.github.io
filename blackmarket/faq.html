Axion Black Market - FAQ

Frequently asked questions and troubleshooting for Axion Black Market

General Questions

What frameworks are supported?

Axion Black Market supports the most popular FiveM frameworks:

  • ESX - Full compatibility with ESX framework
  • QBCore - Full compatibility with QBCore framework
  • Standalone - Can work without frameworks (limited functionality)

Simply configure the framework in your config.lua file:

Config.Framework = "esx" -- or "qb-core" or "standalone"

Why aren't shops spawning?

If shops aren't spawning, check these common issues:

  • Timing configuration - Check if shops are set to spawn during current time
  • Location validity - Ensure all coordinates in Config.ShopLocations are valid
  • Dependencies - Verify ox_target and ox_lib are installed and running
  • Console errors - Check server console for error messages
  • Resource status - Ensure the resource is started without errors

How do I add custom items?

To add custom items to the black market:

  1. Edit shop_items.lua - Add your items to the appropriate category
  2. Configure item properties - Set price, weight, description, etc.
  3. Set stock options - Configure unlimited, fixed, or random stock
  4. Test the items - Restart the resource and test purchases

Troubleshooting

Why can't players purchase items?

If players can't purchase items, check these issues:

  • Framework configuration - Ensure ESX/QBCore is properly configured
  • Payment method - Check if payment type matches player's available funds
  • Item existence - Verify items exist in your database/inventory system
  • Player permissions - Check if player has required permissions
  • Distance check - Ensure player is close enough to the shop

UI not appearing

If the shop UI isn't appearing:

  • Ensure ox_target is working
  • Check if ox_lib is properly configured
  • Verify NUI files are in correct location
  • Check browser console (F12) for JavaScript errors

Script not starting

If the script won't start:

  • Check if all dependencies are installed
  • Verify the resource name in server.cfg
  • Check console for error messages
  • Ensure proper file permissions
