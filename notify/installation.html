Installation Steps

1

Purchase and Download

Purchase the script from our Tebex store and download the files:

Get Axion Notify

Purchase the script to get access to download links and support

Purchase Now
2

Install the Script

Place the downloaded files in your resources folder:

resources/
└── axion-notify/
    ├── fxmanifest.lua
    ├── config.lua
    ├── client/
    │   ├── client.lua
    │   └── shared/
    │       └── client.lua
    ├── server/
    │   ├── server.lua
    │   └── shared/
    │       └── server.lua
    └── html/
        ├── index.html
        ├── style.css
        └── script.js
3

Add to Server Config

Add the resource to your server.cfg:

# Axion Notify
ensure axion-notify
4

Configure Notifications

Edit config.lua to customize the notification system:

Config = {}

-- General Settings
Config.DefaultDuration = 5000 -- Default notification duration in milliseconds
Config.MaxNotifications = 5 -- Maximum number of notifications visible at once
Config.Position = 'top-right' -- Position: 'top-right', 'top-left', 'bottom-right', 'bottom-left'
Config.AnimationDuration = 300 -- Animation duration in milliseconds
Config.SoundEnabled = true -- Enable/disable sound effects

-- Notification Types
Config.Types = {
    ['success'] = {
        title = 'Success',
        color = '#2ecc71',
        icon = 'fas fa-check-circle',
        sound = 'success'
    },
    -- ... more types
}
5

Start the Resource

Restart your server or start the resource manually:

# In server console
restart axion-notify

# Or start if not running
start axion-notify

Troubleshooting

Notifications not appearing

  • Check if the resource is started without errors
  • Verify HTML/CSS/JS files are in the correct location
  • Check browser console (F12) for JavaScript errors

Sounds not playing

  • Check if Config.SoundEnabled = true
  • Verify sound files exist in the correct location
  • Check browser permissions for audio playback

Integration issues

  • Verify all required resources are installed and started
  • Check if the resource names match exactly
  • Ensure proper event handling and exports

Performance issues

  • Check for infinite loops in your custom code
  • Verify wait times in threads are appropriate
  • Monitor resource usage in the server console
