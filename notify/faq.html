Axion Notify - FAQ

Frequently asked questions and troubleshooting for Axion Notify

General Questions

What frameworks are supported?

Axion Notify supports the most popular FiveM frameworks:

  • ESX - Full compatibility with ESX framework
  • QBCore - Full compatibility with QBCore framework
  • Standalone - Can work without frameworks

Simply configure the framework in your config.lua file:

Config.Framework = "esx" -- or "qb-core" or "standalone"

Why aren't notifications appearing?

If notifications aren't appearing, check these issues:

  • Resource status - Ensure axion-notify is started without errors
  • UI files - Check if HTML/CSS/JS files are in the correct location
  • NUI permissions - Verify NUI is properly configured in fxmanifest.lua
  • Console errors - Check browser console (F12) for JavaScript errors
  • Position settings - Check if notifications are positioned off-screen

How do I customize notification appearance?

To customize notification appearance:

  1. Edit config.lua - Modify colors, icons, and durations
  2. Customize CSS - Edit the HTML/CSS files for advanced styling
  3. Add custom types - Create new notification types with custom properties
  4. Modify animations - Adjust animation duration and effects

Troubleshooting

Why are sounds not playing?

If notification sounds aren't playing:

  • Sound enabled - Check if Config.SoundEnabled = true
  • Sound files - Ensure sound files exist in the correct location
  • Browser permissions - Check if browser allows audio playback
  • Volume settings - Verify game and browser volume levels
  • Sound format - Ensure sound files are in supported format

How do I integrate with other scripts?

To integrate Axion Notify with other scripts:

  • Use exports - Call the notification functions from other resources
  • Override framework notifications - Replace ESX/QBCore notifications
  • Use events - Trigger notifications via events
  • Custom integration - Modify other scripts to use Axion Notify

Performance issues

If you're experiencing performance issues:

  • Check for infinite loops in your custom code
  • Verify wait times in threads are appropriate
  • Monitor resource usage in the server console
  • Optimize database queries and network calls
