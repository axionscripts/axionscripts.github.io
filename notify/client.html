Axion Notify - Client Customization

Learn how to customize Axion Notify with client-side modifications

Client-Side Overview

Axion Notify provides a shared/client.lua file that allows you to add custom client-side functionality. This file is designed to be easily customizable without modifying the core script files.

Important Note

The shared/client.lua file is loaded after the main client files, allowing you to override or extend existing functionality without breaking updates.

File Structure

axion-notify/
├── client/
│   ├── client.lua          # Main client file (DO NOT EDIT)
│   └── shared/
│       └── client.lua      # Your custom client code (EDIT THIS)
├── server/
├── html/
├── config.lua
└── fxmanifest.lua

Custom Notification Types

Add custom notification types and behaviors:

Example: Custom Notification Types

-- axion-notify/client/shared/client.lua

-- Add custom notification types
local customTypes = {
    ['police'] = {
        title = 'Police Alert',
        color = '#3498db',
        icon = 'fas fa-shield-alt',
        sound = 'police_alert',
        duration = 8000
    },
    ['gang'] = {
        title = 'Gang Message',
        color = '#e74c3c',
        icon = 'fas fa-skull',
        sound = 'gang_notification',
        duration = 6000
    },
    ['admin'] = {
        title = 'Admin Message',
        color = '#f39c12',
        icon = 'fas fa-crown',
        sound = 'admin_notification',
        duration = 10000
    }
}

-- Override the notification function to include custom types
local originalNotify = exports['axion-notify']:notify
exports['axion-notify']:notify = function(message, type, duration)
    if customTypes[type] then
        local customType = customTypes[type]
        return originalNotify(message, type, duration or customType.duration)
    end
    return originalNotify(message, type, duration)
end

Custom Sound Integration

Add custom sound effects for different notification types:

Example: Custom Sound System

-- axion-notify/client/shared/client.lua

-- Custom sound system
local customSounds = {
    ['success'] = 'SUCCESS_SOUND',
    ['error'] = 'ERROR_SOUND',
    ['warning'] = 'WARNING_SOUND',
    ['info'] = 'INFO_SOUND',
    ['police'] = 'POLICE_ALERT',
    ['gang'] = 'GANG_NOTIFICATION'
}

-- Override the notification function to include custom sounds
local originalNotify = exports['axion-notify']:notify
exports['axion-notify']:notify = function(message, type, duration)
    -- Play custom sound
    if customSounds[type] then
        PlaySoundFrontend(-1, customSounds[type], "HUD_AWARDS", 1)
    end
    
    -- Add custom visual effects
    if type == 'error' then
        SetFlash(0, 0, 100, 500, 100)
    elseif type == 'success' then
        SetFlash(0, 100, 0, 500, 100)
    end
    
    return originalNotify(message, type, duration)
end

Best Practices

Code Organization

  • Keep your custom code organized and commented
  • Use descriptive function and variable names
  • Group related functionality together
  • Add error handling for all custom functions

Error Handling

  • Always check if resources are available before using them
  • Use pcall() for potentially failing operations
  • Provide fallback functionality when possible
  • Log errors for debugging purposes

Performance

  • Avoid creating threads that run constantly
  • Use appropriate wait times in loops
  • Clean up resources when no longer needed
  • Optimize database queries and network calls
